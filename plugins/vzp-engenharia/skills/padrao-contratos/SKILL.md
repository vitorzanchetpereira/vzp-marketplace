# Padrão de contratos — VZP / Base Empreendimentos

Padrão definido pela equipe jurídica/incorporação (contato: Rosana — grupo
"INC - CONST - JUR - BASE EMPREENDIMENTOS").

## Qualificação das partes no preâmbulo

Qualificação completa de PJ / administrador, **nesta ordem**:

1. Razão social
2. "pessoa jurídica de direito privado, inscrita sob o CNPJ nº ..."
3. Endereço da sede — **CEP sempre antes da cidade**
   Ex.: "...CEP 78890-148, na cidade de Sorriso, Estado de Mato Grosso"
4. Representante: "neste ato representada pelo administrador não sócio [Nome],
   brasileiro, [estado civil], [profissão], inscrito no CPF sob o nº ...,
   portador do RG nº ... SESP/MT, residente e domiciliado na [endereço
   residencial], CEP ..., na cidade de ..., Estado de ..."

**Regra prática (enfatizada pelo jurídico):** "TOC no CEP, sempre antes da cidade."

## Revisão de contrato

- Ao incorporar parecer jurídico, registrar o **histórico de disposição de cada
  item** (acatado / não acatado e por quê).
- Entregar a revisão em **controle de alterações** (redline / tracked changes),
  sob autor identificável, para que cada mudança seja conferível na própria minuta.
- Nem todo item do parecer é acatado — confirmar com a CONTRATANTE itens sensíveis
  (ex.: Propriedade Intelectual, fluxo fiscal) antes de alterar.
- **Numeração de cláusula nova:** o padrão do escritório é renumerar todo o
  restante quando se insere uma cláusula inline. Em redline enviado à contraparte,
  é aceitável inserir com numeração *bis* (ex.: "Cláusula Quadragésima Quinta-A")
  para não poluir o documento com renumeração de todas as cláusulas seguintes;
  ao consolidar a versão final limpa, renumerar de forma corrida. Seguir o que o
  parecer jurídico indicar e confirmar com a Rosana.

## Parecer de Alterações (documento auxiliar obrigatório por revisão)

**Toda revisão de contrato sai acompanhada de um documento auxiliar** que lista as
alterações daquela revisão, para enviar JUNTO com a minuta a quem vai assinar — o
signatário precisa saber exatamente o que mudou.

- É da **família "Parecer"** do escritório. Escolher o nome conforme o estágio:
  `...-ParecerAnalise-R##` (análise da minuta recebida) · `...-ParecerAlteracoes-R##`
  (histórico das alterações aplicadas na revisão) · `...-ParecerLiberacao-R##`
  (libera a minuta para assinatura) · `...-ParecerConformidadeAssinatura-R##`
  (confere a via assinada contra a versão aprovada).
- **Estrutura padrão:** timbre BASE (logo + "DOCUMENTO"/código no topo à direita
  + régua vermelha BE1E2D); título vermelho; bloco-resumo das partes (CONTRATANTE,
  CONTRATADA, OBJETO/VALOR, DOCUMENTO base, ORIGEM); Objetivo/Veredito; **tabela de
  histórico de alterações** com colunas Nº · Cláusula · Alteração · Tipo
  (Retificação / Complementação / Inclusão) — ou Origem (Parecer jurídico / Parecer
  análise / Decisão comercial + nome / Jurídico–Rosana); seção destacando os pontos
  que alteram a lógica do contrato; Encaminhamento; rodapé
  "BASE Empreendimentos Imobiliários Ltda. | ... | Página X de Y". Fonte Arial,
  cinza 726E67 nos textos secundários.
- Salvar o parecer na **mesma pasta da obra** da minuta (ex.:
  `.../02-Contratos/<disciplina>/`), com o mesmo prefixo de data e o mesmo R## da
  revisão. Enviar ao signatário o parecer + a minuta em controle de alterações
  (e a versão limpa, quando houver).

## Dados das partes

Os dados de cada SPE/empreendimento (CNPJ, endereço, administrador, CPF/RG) ficam
com o setor jurídico. Pergunte ao usuário ou consulte a ficha da obra ao preencher
o preâmbulo — não presuma dados pessoais.

Arquivar contratos conforme a skill **arquivamento-obras**; se registrar no Asana,
o responsável é a Duana (skill **suprimentos-asana**).
